# 타락한 객체
- 일반 적인 TDD 주기
    - 테스트를 작성
        - 오퍼레이션이 코드에 어떻게 나타나길 원하는지
        - 원하는 인터페이스
    - 실행 가능하게 만든다
        - 빨리 초록 막대(테스트 성공)를 보도록
    - 올바르게 만든다
        - 빠르게 만들었던 죄악을 수습
        - 중복을 제거하고 초록 막대로
- 작동하는 깔끔한 코드를 만드는 것은 어렵다
    - 동시에 해결하려고 하지말고 먼저 '작동하는'을 해결
- 최대한 빠른 초록색
    - 가짜로 구현하기
    - 명백한 구현 사용하기
